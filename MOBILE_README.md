# 스마트폰에서 재고 확인 프로그램 쓰는 방법

현재 프로그램은 **Playwright(실제 브라우저)**를 사용해서 티파니 사이트를 열고, 클릭·스크롤까지 해서 슬라이드 패널 내용을 가져옵니다.  
그래서 **스마트폰 안에서 이 프로그램을 그대로 돌리는 것**은 거의 불가능에 가깝고, 대신 아래 방식들을 쓸 수 있습니다.

---

## 1) 추천: 클라우드에서 실행 → 스마트폰에서 메일만 받기

**프로그램은 PC/스마트폰이 아니라 클라우드에서 돌리고**, 재고 있으면 **스마트폰 메일함으로 알림**이 오게 하는 방식입니다.

- **장점**: 스마트폰 전원/배터리와 무관, 24시간 스케줄 가능, 기존 코드 그대로 사용
- **단점**: GitHub/클라우드 계정·설정 필요

**예: GitHub Actions (무료)**

1. 이 프로젝트를 GitHub 저장소에 올립니다.
2. 저장소 **Settings → Secrets**에  
   `TIFFANY_SEND_EMAIL`, `TIFFANY_RECEIVE_EMAIL`, `TIFFANY_APP_PASSWORD` 를 넣습니다.
3. **Actions** 탭에서 “매일 오전 10시(한국 시간)”에 `monitor_size_5_8.py --once` 를 실행하는 워크플로를 만듭니다.
4. 재고가 있으면 Gmail로 발송 → **스마트폰에서 메일 앱으로 확인**하면 됩니다.

이렇게 하면 “스마트폰에서 프로그램이 돌아가는” 건 아니지만, **결과는 스마트폰에서 메일로 받는** 형태로 사용할 수 있습니다.

---

## 2) Android: Termux로 “폰에서 직접” 실행 시도 (고급)

**Android**에서는 **Termux**로 Python을 깔고, 같은 로직을 돌릴 수는 있지만 제약이 많습니다.

### 제약

- 현재 코드는 **Playwright + Chromium**이 필요합니다.  
  Termux에서 Playwright/브라우저 설치가 가능한 경우도 있지만, 설정이 복잡하고 무겁습니다.
- **배터리·절전** 때문에 백그라운드에서 매일 10시에만 깨워서 실행하는 건 **Tasker + Termux 플러그인** 등 추가 설정이 필요합니다.
- **iOS**에서는 Termux 같은 환경이 없어서, 같은 방식으로 “폰에서 직접 실행”하는 건 사실상 불가능합니다.

### Termux에서 해볼 수 있는 것

1. **Termux** 설치 (F-Droid 또는 공식 사이트)
2. `pkg update && pkg install python` 후 `pip install playwright`  
   → `playwright install chromium` 은 Termux에서 실패하거나 매우 무거울 수 있음
3. **경량 버전**을 쓰는 방법:  
   사이트가 **JavaScript 없이** HTML만으로 재고 정보를 준다면, `requests` + HTML 파싱으로 재고만 확인하는 짧은 스크립트를 만들어 Termux에서 돌릴 수 있습니다.  
   **티파니는 슬라이드 패널이 JS로 뜨기 때문에**, 이 경량 방식은 **동작하지 않을 가능성이 큽니다.**  
   그래도 “폰에서 한 번 돌려보고 싶다”면, `monitor_size_5_8.py` 대신 **requests만 쓰는 스크립트**를 따로 만들어 시험해 볼 수는 있습니다 (성공 보장은 어렵습니다).

정리하면, **Android에서도 “실제로 잘 돌리려면”**  
- 클라우드(GitHub Actions 등)에서 기존 프로그램 실행 + 스마트폰에서 메일 받기  
를 추천하고,  
- Termux는 “고급 사용자가 폰에서 직접 실행해 보는 용도”로만 생각하는 게 좋습니다.

---

## 3) iOS

- iOS에서는 PC처럼 Python/Playwright를 설치해서 스크립트를 돌리는 환경이 없습니다.
- 따라서 **스마트폰에서 해당 프로그램이 돌아가게** 하는 방법은 **없고**,  
  **클라우드에서 실행 → 스마트폰에서 메일/알림으로 받기**만 가능합니다.

---

## 요약

| 목표                         | 방법 |
|-----------------------------|------|
| 스마트폰에서 **결과만** 받고 싶다 | **클라우드(GitHub Actions 등)**에서 매일 실행 + **스마트폰 메일**로 수신 (추천) |
| Android에서 **직접** 실행해 보고 싶다 | **Termux** + Python (Playwright는 무겁고, 경량 버전은 티파니에서 동작 안 할 수 있음) |
| iOS에서 프로그램 돌리기     | 불가. 클라우드 실행 + 메일 받기만 가능 |

**실질적으로 “스마트폰에서 해당 프로그램이 돌아가게”** 쓰려면,  
**클라우드에서 돌리고 → 스마트폰은 알림(메일)만 받는 기기**로 쓰는 구성이 가장 현실적입니다.

GitHub Actions로 “매일 10시 KST 실행 + 재고 시 메일” 설정 예시가 필요하면 요청해 주세요.
